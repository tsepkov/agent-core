# Вводное описание проекта: ИИ-управляющий для Ozon

**Название концепта:** Автономный ИИ-агент управления маркетплейсом.

### Суть проекта
Проект представляет собой интеллектуального автономного помощника для продавцов на Ozon, который берет на себя рутинные и аналитические процессы: управление ценами, оптимизацию контента, предотвращение кассовых разрывов через глубокий аудит маржинальности и сквозную координацию поставок FBO. 

Главное отличие от классических сервисов аналитики — агент не просто показывает графики, а действует: общается в чате, генерирует контент, находит «протечки» прибыли, рассчитывает поставки и сам идет в Ozon API выполнять задачи.

---

### Архитектурный фундамент
Система спроектирована под стек **Restate + Vercel AI SDK** и разворачивается в среде **Vercel Serverless**. 

Использование принципа Durable Execution (надежного прерывистого исполнения) полностью решает проблему ограничений serverless-архитектуры. Агентские циклы могут длиться часами и днями: система умеет засыпать во время долгих генераций медиа-контента или в ожидании ответа пользователя, не расходуя лишние вычислительные ресурсы, и просыпаться ровно на том шаге, где остановилась, со стопроцентным сохранением операционного контекста.

---

### Ключевые возможности системы
1. **Каждый запрос к API — инструмент:** Любое действие с Ozon API (чтение остатков, изменение цен, создание документов) или внешними сетями генерации графики/видео изолировано в виде атомарного инструмента агента.
2. **Проактивность и автономность:** Агент работает не только по команде из чата, но и самостоятельно просыпается по заданному интервалу времени для проведения ревизии личного кабинета.
3. **Безопасность (Human-in-the-Loop):** Все критические финансовые или контентные изменения (изменение цен, применение инфографики, подтверждение слотов FBO) требуют явного апрува пользователя через интерфейс.
4. **Гибридная память:** Высокая скорость и экономия контекста LLM за счет хранения оперативных данных внутри стейта Restate, в сочетании с обособленной долгосрочной семантической памятью во внешней базе данных.
5. **Интерактивный Web UI:** Современный фронтенд на Next.js, который стримит «мысли» (Reasoning) агента и выводит результаты работы инструментов в виде готовых интерактивных графиков, виджетов и форм подтверждения действий.

## 1. Концепт и Стек
* **Суть:** Автономный ИИ-агент для продавцов на Ozon (управление ценами, контентом, аналитикой прибыли и поставками FBO).
* **Стек:** Restate (Durable Execution) + Vercel AI SDK + Vercel Serverless (среда выполнения) + Next.js (фронтенд).
* **Связь стека:** Агентский цикл работает на Serverless. Restate гарантирует, что шаги воркфлоу, долгие генерации и вызовы API Ozon застрахованы от таймаутов, а при сбоях выполнение возобновляется ровно с места остановки без повторения уже сделанных шагов.

## 2. Глобальные требования к логике и архитектуре

### Инструменты (Каждый запрос/группа запросов к API — это отдельный инструмент):
* **Контент и цены:** Изменение цен, участие в акциях, редактирование SEO и SKU-параметров, асинхронный запуск генерации инфографики/видео через внешние API и их загрузка в карточки.
* **Аналитика маржинальности:** Сбор сырых отчетов по продажам, оборачиваемости и остаткам для автоматического поиска «протечек» прибыли.
* **Поставки FBO:** Расчет потребности в завозе, поиск доступных таймслотов/кросс-докинга, создание черновика заявки на поставку в ЛК Ozon.

### Режимы работы:
* **Интервальный (Proactive):** Агент сам просыпается по заданному расписанию, вызывает аналитические инструменты, проводит аудит кабинета и готовит решения.
* **Ассистент (Reactive):** Точечно отвечает на команды в чате.
* **Human-in-the-Loop:** Для критических действий (цены, медиа, отправка поставок) агент замораживает воркфлоу и ждет явного апрува от пользователя.

### Хранение данных и память:
* **Оперативный контекст:** История диалога, промежуточная аналитика и текущий стейт накапливаются и удерживаются внутри долгоживущего состояния Restate (без привлечения внешней БД).
* **Долгосрочная память:** Выводы, профиль продавца и глобальные правила («не продавать товар X ниже Y рублей») выносятся во внешнюю базу данных через отдельный обработчик.

### Отказоустойчивость:
* Логика использует механизмы Restate для последовательной обработки задач конкретного личного кабинета маркетплейса во избежание Rate Limits. Повторы при ошибках API настраиваются на уровне шагов Restate. Все операции записи защищены токенами идемпотентности.

### Веб-интерфейс:
* Чат со стримингом мыслей (Reasoning) в реальном времени и Generative UI (выдача результатов инструментов в виде виджетов, графиков и таблиц слотов). Панель жизненного цикла воркфлоу (статус агента, ручной запуск аудита, сброс контекста) и Центр подтверждений с кнопками для Human-in-the-Loop.

## 3. Поэтапный план разработки (Разбивка по сложности)

### Шаг 1: Минимальный жизнеспособный агент (MVP Loop)
* **Интерфейс:** Окно чата (Next.js) с потоковой передачей текста и выводом мыслей (Reasoning) агента в реальном времени.
* **Логика и сессия:** Базовый воркфлоу-цикл агента, удерживающий в оперативной памяти историю текущей беседы в рамках одной сессии.
* **Единственный инструмент:** `getMarketplaceStoreInfo` — запрос базовой информации о подключенном магазине (название, статус, ID) из API Ozon.

### Шаг 2: Асинхронные инструменты и генерация медиа
* **Асинхронный инструмент:** Инструмент запуска генерации картинок/видео через стороннее API.
* **Механика ожидания:** Агент отправляет запрос наружу и «засыпает» (scale to zero), ожидая вебхука от генератора медиа для продолжения цикла.
* **UI-виджеты:** Результат рендерится в чате интерактивным UI-компонентом сравнения картинок «До / После».

### Шаг 3: Автономность и Human-in-the-Loop
* **Интервальный запуск:** Агент просыпается сам по таймеру и запускает фоновый аудит.
* **Точка прерывания:** Добавление инструмента записи (например, изменение цены). При вызове воркфлоу замораживается до подтверждения.
* **Интерфейс апрувов:** Физические кнопки «Одобрить действие» / «Отклонить» в UI для отправки сигнала агенту.

### Шаг 4: Глубокая аналитика и инструменты FBO
* **Инструменты аналитики:** Группа инструментов для вытягивания сырых отчетов Ozon и их агрегации в памяти сессии.
* **Инструменты FBO:** Инструменты проверки остатков на складах, поиска свободных слотов кросс-докинга и создания черновика поставки в ЛК Ozon.

### Шаг 5: Долгосрочная память и семантический контекст
* **Вынос памяти:** Интеграция с внешней базой данных. Сохранение глобальных выводов и правил торговли между сессиями.
* **Управление лимитами:** Тонкая настройка контроля частоты вызовов инструментов для жесткого соблюдения Rate Limits Ozon API.
